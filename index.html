HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

The 3 types of CSS

  • Inline: used to style a specific, single element directly in the HTML file.
  • Internal:used to style an entire stylesheet directly in the HTML file using a style element in the head element (meta code)
  • External: uses a seperate, linked file (linked in the HTML file's head) and is the most common/preferred method

Terms and Commands

  • Margin: space outside of an element, not visable on the page. Increasing value adds a buffer, 0 value indicates NO extra space around the element
  • Padding: space inside the element. Increasing value makes the entire element bigger.
  • * symbol: used to apply a rule to ALL the elements visible on the page
  • Link External CSS: in the head section of HTML file, after the final META but before the TITLE
  • Rel: specifies the relationship between current doc (ie the HTML file) and the linked doc (ie external CSS stylesheet)
  • Href: specifies the location (URL) of the external, linked resource and always starts with ./ (ie ./assets/style.css)

Concepts and Framework

  • CSS Box Model: every HTML element is a box. Each boxed element is surrounded by another box element, as shown below
  • DRY Code: Do Not Repeat Yourself encourages reusing/shairng code to reduce # of lines that need to be written or maintained. Keeps code as simple and small as possible

Git

Commands

  • git checkout -b branch-name: creates a new branch and switches to working on it
  • git status: checks what branch you're currently working on
  • git add -A: stages changes to be committed
  • git commit -m "name/details of commit": commits changes
  • git pull/git pull (branch name): pulls from Github repo
  • git push/git push (branch name): pushes commits to Github repo

JavaScript

Terms and Commands

  • Var: declares a variable, is followed by the unique variable name that can be referenced
  • = symbol: assignment operator, assigns specific data to the command
  • Array: a single variable that is used to hold a group of (usually related) data. Can store multiple items with different data types in a single variable
  • String: must be enclosed in quotation marks
  • Index: unique number (starting at 0) used to identify individual data items and their location inside an array
  • For Loop: uses the predictable pattern of an index to perform a task on all the items in an array by executing a single code block again and again. Contains the 3 following statements:
    • Starting Point: [var x = 0] declare a variable (x) and give it a value (0, to start in the begining becasue the firxt index of an array is always 0)
    • Condition: As long as the number in the variable (x) is less than the length of the array the loop will keep running. Use the built-in LENGTH property to express
    • Repetition: [x++] to allow the code to loop over each item in the array
  • Function: [function NAME () {CODE BLOCK}] set of instructions that do NOT automatically execute when the JavaScript file is run and must be called by name to execute

Concepts and Framework

  • Control Flow: the order that code is executing in a script
